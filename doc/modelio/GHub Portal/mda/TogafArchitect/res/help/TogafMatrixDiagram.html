Matrix

The purpose of Matrix diagrams is to define TOGAF dependencies that will produce the TOGAF Matrixes. Matrix diagrams are also useful to define general purpose traceabilities between elements.

Under Modelio: Drag and drop in the diagram the origin and destination elements, and trace the appropriate links.

- For Business Interaction Matrixes : use "participates", "consumes" or "support" links

- For Actor/Role Matrixes : use "assumes" links

- For Data Entity/Business Function Matrixes : use trace links from data entities to functions

- For System/Data Matrixes : use flow links between Application Components and Business Entities

- For System/Organization Matrixes : use "consumes" links

- For Role/System Matrixes : use "consumes" links

- For System/Function Matrixes : use "realizes" links

- For Application Interaction Matrixes : use imports between components, or connectors between Services Accesses
